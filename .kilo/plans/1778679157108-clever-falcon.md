# Search Dropdown & Globe Location Accuracy Standardization

## Problem Summary

Two distinct problems:

1. **Search dropdown** — `PlaceSuggestionList` renders primary/secondary text on a single flex row (`justify-content: space-between`), causing truncation, poor contrast, and layout instability. The `position: absolute` panel can be clipped by `overflow: hidden` on parent wrappers.

2. **Globe rotation math** — `targetRotationFromLatLng()` computes incorrect Euler angles. For `lat=0, lng=0`, the 3D point is at `(1, 0, 0)` (+X axis), but the function returns `{x:0, y:0}`, leaving the point on the right side instead of facing the camera at +Z. The fly-to effect also lacks shortest-path wrap handling, causing 350° spins instead of 10° the other way.

## Step 2: Fix Coordinate System — Single Source Functions

**File**: `src/features/globe/geometry.ts`

### Current problem

`targetRotationFromLatLng` uses `x = phi - PI/2`, `y = -theta + PI` which doesn't produce the correct Euler angles to face a given lat/lng toward the camera at +Z.

The correct derivation: given a 3D point P = (px, py, pz) on the unit sphere, find Euler angles (rx, ry) in XYZ order such that `Rx(rx) * Ry(ry) * P = (0, 0, R)`.

From the math:
```
ry = atan2(-px, pz)
z' = -px * sin(ry) + pz * cos(ry)
rx = atan2(py, z')
```

### New `geometry.ts`

```ts
import * as THREE from 'three';

export const TEXTURE_Y_OFFSET = 0; // calibrated empirically; adjust if texture is shifted

export function latLngTo3D(lat: number, lng: number, radius: number): THREE.Vector3 {
  const phi = (90 - lat) * (Math.PI / 180);
  const theta = (lng + 180) * (Math.PI / 180);

  const x = -(radius * Math.sin(phi) * Math.cos(theta));
  const y = radius * Math.cos(phi);
  const z = radius * Math.sin(phi) * Math.sin(theta);

  return new THREE.Vector3(x, y, z);
}

export function targetRotationFromLatLng(lat: number, lng: number): { x: number; y: number } {
  const pos = latLngTo3D(lat, lng, 1);

  const ry = Math.atan2(-pos.x, pos.z);
  const zPrime = -pos.x * Math.sin(ry) + pos.z * Math.cos(ry);
  const rx = Math.atan2(pos.y, zPrime);

  return {
    x: rx,
    y: ry + TEXTURE_Y_OFFSET,
  };
}

export function shortestAngleDelta(from: number, to: number): number {
  let delta = to - from;
  while (delta > Math.PI) delta -= 2 * Math.PI;
  while (delta < -Math.PI) delta += 2 * Math.PI;
  return delta;
}
```

Key changes:
- `latLngTo3D` stays unchanged (it's internally consistent)
- `targetRotationFromLatLng` now uses `atan2` derived from the 3D position vector, guaranteeing the point faces +Z after rotation
- `TEXTURE_Y_OFFSET` allows calibration if the earth texture is rotated relative to the math model
- New `shortestAngleDelta` utility for fly-to shortest path

## Step 3: Fix Fly-To with Shortest-Path Rotation

**File**: `src/features/globe/GlobeScene.tsx` (fly-to effect)

Replace the current linear interpolation with shortest-path for Y rotation:

```ts
useEffect(() => {
  if (!targetRotation || !sceneRef.current) return;

  const startX = rotationX.current;
  const startY = rotationY.current;
  const endX = Math.max(-MAX_PITCH, Math.min(MAX_PITCH, targetRotation.x));
  const endY = targetRotation.y;

  const deltaY = shortestAngleDelta(startY, endY);
  const actualEndY = startY + deltaY;

  const duration = 1500;
  const startTime = performance.now();
  autoRotate.current = false;

  function animateFly(now: number) {
    const elapsed = now - startTime;
    const progress = Math.min(elapsed / duration, 1);
    const eased = 1 - Math.pow(1 - progress, 3);

    rotationX.current = startX + (endX - startX) * eased;
    rotationY.current = startY + deltaY * eased;

    if (progress < 1) {
      requestAnimationFrame(animateFly);
    } else {
      rotationY.current = actualEndY;
      setTimeout(() => { autoRotate.current = true; }, 2000);
      onFlyComplete?.();
    }
  }

  requestAnimationFrame(animateFly);
}, [targetRotation, onFlyComplete]);
```

**File**: `src/features/globe/useGlobeController.ts`

Simplify to use the corrected `targetRotationFromLatLng`:

```ts
import { useState, useCallback, useRef } from 'react';
import type { FlyToCoordsInput, GlobeController } from './globeTypes';
import { targetRotationFromLatLng } from './geometry';

export function useGlobeController(): {
  targetRotation: { x: number; y: number } | null;
  controller: GlobeController;
} {
  const [targetRotation, setTargetRotation] = useState<{ x: number; y: number } | null>(null);
  const animatingRef = useRef(false);

  const flyToCoords = useCallback((input: FlyToCoordsInput) => {
    if (animatingRef.current) return;

    setTargetRotation(targetRotationFromLatLng(input.lat, input.lng));

    animatingRef.current = true;
    setTimeout(() => {
      animatingRef.current = false;
    }, input.duration ?? 1500);
  }, []);

  return {
    targetRotation,
    controller: { flyToCoords },
  };
}
```

Key change: delegates rotation computation to `targetRotationFromLatLng` in `geometry.ts` instead of duplicating phi/theta math.
